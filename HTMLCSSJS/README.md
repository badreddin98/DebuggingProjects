# HTML, CSS, and JavaScript Debugging Exercises

This repository contains debugging exercises for HTML, CSS, and JavaScript. Each exercise is designed to help students identify and fix common issues in web development.

## HTML Debugging Exercise

The HTML debugging exercise (`HTML/html-debugging/index.html`) contains 10 broken scenarios that violate HTML best practices:

1. Missing alt attributes for images
2. Improper heading hierarchy
3. Non-semantic div usage
4. Missing form labels
5. Improper table structure
6. Missing ARIA attributes
7. Improper list usage
8. Missing form validation
9. Improper link usage
10. Missing meta descriptions

## CSS Debugging Exercise

The CSS debugging exercise (`CSS/css-debugging/styles.css`) contains 10 broken scenarios that violate CSS best practices:

1. Inline styles instead of CSS classes
2. Non-responsive units
3. Overly specific selectors
4. Missing vendor prefixes
5. Improper box model
6. Non-semantic color values
7. Missing media queries
8. Improper z-index usage
9. Missing fallback fonts
10. Non-accessible color contrast

## JavaScript Debugging Exercise

The JavaScript debugging exercise (`JavaScript/js-debugging/script.js`) contains 10 broken scenarios that violate JavaScript best practices:

1. Global variable pollution
2. Callback hell
3. Improper error handling
4. Memory leak
5. Race condition
6. Improper type checking
7. Synchronous blocking code
8. Improper closure usage
9. Improper promise handling
10. Improper object mutation

## How to Run the Exercises

1. Clone the repository
2. Open each exercise file in your preferred code editor
3. Identify the issues in each scenario
4. Fix the issues following best practices
5. Test your solutions

## Learning Objectives

- Understand HTML semantics and accessibility
- Learn CSS best practices and responsive design
- Master JavaScript modern practices and async programming
- Develop debugging skills in web development
- Apply best practices in real-world scenarios
